UI

Color as Information

stroy uses color to tell about differences.
  • black file unchanged
  • green file content changed
  • blue name and/or parent changed
  • red this file / directory exists only in this tree
  • light blue file content and name and / or parent changed

Tooltips and Infobar

Tooltips summerize the changes of the file or directory. The later is a bit more interesting. The window shows the full path of both files on the bottom in an extra info bar.

Skyview Bars

The colored strips aside the scrollbars are clickable. The colored blocks represent areas of change in the tree. Because of size constraints the summerize several changes. A click on a colored block jumps to the first of these.

Close is Ok

All dialogs without an ok button act on the choice either imediately or on close. The revert the choice click reset.
